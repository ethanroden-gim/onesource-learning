Module 7

Responding to Board Questions Without Overstepping

Learning Objective

By the end of this module, you will be able to use a consistent response framework to answer basic board questions about Buildium financial packages and identify when finance should confirm the accounting detail.

What you will learn

Module focus

  • How to structure a clear financial response
  • How to connect a board request to the right report
  • How to give factual answers without guessing
  • How to escalate accounting interpretation professionally
GIM standard

Clear, accurate, transparent

Boards should feel supported and informed. PMs should explain what Buildium shows, stay within the report, and bring finance in when a question requires accounting judgment.

Key idea

The strongest answer is not always the fastest answer. It is the answer that is accurate, report-based, and followed up when finance review is needed.

Framework

Use the same structure every time

A consistent process helps property managers answer basic financial questions clearly while avoiding unsupported accounting explanations.

1. ConfirmRestate the request so the question is clear.
2. IdentifySelect the Buildium report that answers it.
3. ExplainState what the report is designed to show.
4. AnswerGive the factual answer from the report.
5. EscalateSend interpretation, corrections, or accounting treatment to finance.
6. Follow upClose the loop once finance confirms the answer.

Why this matters

Skipping straight to an answer can lead to guessing. The framework keeps the response grounded in the report and protects board trust.

Find

Match the request to the report

Most board questions become easier once you choose the correct report first.

Use these reports for summary questions

  • Balance Sheet: cash, receivables, payables, loans, equity position
  • Income Statement: income, expenses, net income for a period
  • Budget vs. Actuals: performance against the approved budget

Use these reports for detail questions

  • Delinquent Association Owners: owner balances, aging buckets, collections status
  • Transaction Detail by Account: entries behind a total, dates, descriptions, transaction types

PM move

Before answering, name the report: “The report that answers this best is the Budget vs. Actuals because it compares actual results to the approved budget.”

Answer

Explain what Buildium shows, then give the basic answer

Step 3

Explain the report first

  • “The Balance Sheet is a point-in-time report.”
  • “The Income Statement covers this reporting period.”
  • “The Budget vs. Actuals compares actual activity to budget.”
  • “Transaction Detail shows the entries behind the total.”
Step 4

Answer with the report data

  • Use the report date or period.
  • Point to the relevant line item.
  • Use the exact amount shown.
  • Clarify monthly vs. year-to-date when needed.

Buildium phrasing

Use report-based language: “Buildium shows…” or “As of the report date…” instead of language that implies the item has been independently audited or fully reconciled by the PM.

Escalate

Know where the PM answer stops

PM can usually answer

  • What a report is showing
  • Which line item contains the amount
  • How much was recorded
  • What period or date the report covers
  • What transaction detail is listed

Finance should confirm

  • Why an accounting entry was posted
  • Whether a balance is correct
  • Whether an item should be accrued or reclassified
  • Opening balance or prior-period questions
  • Corrections, reversals, or reconciliations

Escalation rule

If the question asks what Buildium shows, the PM can usually explain it. If the question asks why it was booked that way or whether it should change, finance should confirm.

Language

Use wording that is helpful and accurate

Recommended phrases

  • “The report is showing…”
  • “As of the report date…”
  • “For this reporting period…”
  • “The detail behind that total appears on…”
  • “Finance should confirm the accounting treatment.”

Avoid phrases like

  • “I assume…”
  • “It is probably…”
  • “That must mean…”
  • “I am sure that was booked correctly.”
  • “The owner will definitely pay.”

Strong escalation wording

“I can explain what Buildium is showing here. The accounting reason behind that entry should be confirmed by finance, and I’ll get that reviewed.”

Wrap-Up

Answer clearly, escalate early, follow up reliably

PMs should consistently

  • Confirm the question before answering
  • Choose the correct Buildium report
  • Explain the purpose of that report
  • Use exact report dates, periods, line items, and amounts
  • Keep responses factual and board-appropriate

PMs should escalate when asked about

  • Accounting treatment or journal entry purpose
  • Opening balances or prior-period activity
  • Accruals, reclassifications, or reversals
  • Ledger disputes, collection actions, or waivers
  • Whether the report requires correction

Module complete

The goal is not to answer every financial question on the spot. The goal is to answer the basic questions accurately, know when finance needs to confirm, and protect the board’s confidence in GIM’s financial transparency.
